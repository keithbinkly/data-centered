The Big Picture

New year, new paradigm. This batch captures a turning point: the semantic layer debate went mainstream, and the tooling ecosystem for AI-assisted development exploded in December.

Three threads dominate:

  1. The Inmon-Kimball debate gets a sequel — Bill Inmon himself published on the architecture wars, but reframed for the AI age. Systems of record vs. systems of insight is the new dimensional vs. normalized.
  2. Context graphs emerge — Jay Gupta's Twitter thread on "trillion-dollar opportunity" in context graphs captured something in the air. It's not just knowledge graphs anymore—it's graphs of context.
  3. MCP becomes real infrastructure — Model Context Protocol moved from spec to implementation. MotherDuck shipped an MCP server. The plugin ecosystem is forming.

Cross-Cutting Insights

1. The "context graph" framing is catching on. Gupta's thread resonated because it names something practitioners have been feeling: knowledge graphs are too academic, but we need something to represent business context. "Context graph" bridges the gap.

2. Memory is the next frontier. Three separate projects tackling cross-session memory for Claude/AI tools. The pattern: context is expensive, persistence is cheap, so save everything and let retrieval figure it out.

3. MCP is becoming infrastructure. MotherDuck shipping an MCP server is significant—it's not just a protocol anymore, it's a platform assumption.

4. The Inmon-Kimball debate never died. It just changed clothes. "Systems of record vs. systems of insight" is "normalized vs. dimensional" for the AI age. The pendulum is swinging back toward Inmon's vision, but with knowledge graph characteristics.
